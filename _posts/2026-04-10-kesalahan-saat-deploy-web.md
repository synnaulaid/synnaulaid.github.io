# Kesalahan yang Sering Dilakukan Saat Deploy Website (dan Cara Menghindarinya)

Halo barudak Simply Karya! Kali ini kita akan membahas tentang kesalahan yang sering dilakukan saat deploy website, terutama bagi kalian yang baru memulai perjalanan di dunia web development. Deploying a website can be an exciting but also nerve-wracking experience, especially if you're new to it. Banyak developer pemula bahkan Developer yang sudah berpengalaman pun sering melakukan kesalahan yang bisa merusak website mereka. Jadi, mari kita bahas beberapa kesalahan fatal yang sering terjadi saat deploy website dan bagaimana cara menghindarinya.

## Audit Project Sebelum Deploy
Beberapa developer sering terburu-buru untuk segera deploy website mereka tanpa melakukan audit terlebih dahulu. Padahal, melakukan audit project sebelum deploy sangat penting untuk memastikan bahwa semua fitur berjalan dengan baik dan tidak ada bug yang tersembunyi. Pastikan untuk melakukan testing secara menyeluruh, baik itu unit testing, integration testing, maupun end-to-end testing sebelum kamu memutuskan untuk deploy website kamu.

Terkadang developer juga lupa untuk memeriksa dependencies yang digunakan dalam project mereka. Selalu Missconfigure dependencies bisa menyebabkan masalah saat deploy, seperti error yang tidak terduga atau bahkan crash pada website kamu. Jadi, pastikan untuk selalu memeriksa dan memperbarui dependencies kamu sebelum deploy.

Sebaiknya, gunakaan environment staging untuk melakukan testing sebelum deploy ke production. Beberapa developer sering melakukan hal konyol dengan langsung deploy ke production tanpa melakukan testing terlebih dahulu. Hal ini bisa sangat berbahaya karena jika ada bug atau error yang tidak terdeteksi, maka website kamu bisa mengalami downtime atau bahkan kehilangan data penting atau juga bisa di susupi oleh peretas.
